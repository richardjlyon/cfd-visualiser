## Deviations from Plan

### Design Notes

**1. struct.unpack instead of pillow for dimension tests**
- **Rationale:** The plan offered two options: Pillow or struct.unpack on the PNG IHDR chunk. Since pillow is not in pyproject.toml and adding it would be an unnecessary dependency for 3 test assertions, struct.unpack was used instead.
- **Files modified:** tests/test_build_og_image.py
- **Effect:** No extra dependency added; tests pass identically.

**2. Scheduler decision defaulted to option-a without checkpoint pause**
- **Rationale:** Plan Task 1 is `checkpoint:decision` with "Default on no response: option-a." This executor ran in parallel wave mode without user interaction; option-a was applied as the documented default and recorded in docs/OPS.md.
- **Effect:** CI/daily workflows scaffold for GitHub Actions (GitHub-native syntax). If the user prefers option-b or option-c, the workflow files can be adapted — Gitea Actions syntax is a compatible subset.

## Known Stubs

1. **`__CF_WEB_ANALYTICS_TOKEN__`** in `observablehq.config.js` head — intentional placeholder. The real token is substituted into `dist/**/*.html` at deploy time via the sed step in `daily.yml`. If `CF_WEB_ANALYTICS_TOKEN` secret is unset, the placeholder persists and the beacon silently no-ops.

2. **`src/assets/og-card.png`** — not committed here (generated at pipeline run time from `src/data/chart-3c.json`). The daily workflow commits it after each successful run. The CI seed step does not generate it (CI only needs the Framework build to succeed, not the OG card).
